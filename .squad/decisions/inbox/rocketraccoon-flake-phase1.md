## Decisions Made

### 1. Input Sources and Versioning

**Chosen Inputs:**
- `nixpkgs`: `github:nixos/nixpkgs/nixos-unstable`
- `home-manager`: `github:nix-community/home-manager/release-24.05` (pinned to release branch for stability)
- `sops-nix`: `github:Mic92/sops-nix` (latest via flake.lock)

**Rationale:**
- nixpkgs unstable provides latest packages while maintaining stability via lock file
- home-manager release-24.05 ensures compatibility with our home-manager approach
- All inputs follow `inputs.nixpkgs.follows = "nixpkgs"` to maintain version consistency
- Lock file generated by `nix flake update` ensures reproducibility across all machines

**Alternative Considered:**
- Using nixos-24.05 stable for nixpkgs: Rejected because unstable + lock file provides latest tooling while maintaining reproducibility

---

### 2. Output Structure and Helper Function

**Chosen Approach:**
```nix
mkHostConfig = hostname: hostConfig: nixpkgs.lib.nixosSystem { ... }
```

**Why This Matters:**
- Single `mkHostConfig` function eliminates duplication across three machine configs
- Each machine's output is `nixosConfigurations.{hostname}` (per NixOS flake standard)
- `specialArgs` passes `inputs` and `self` to all modules for maximum flexibility
- home-manager integration at system level (nixosModules.home-manager) ensures proper setup

**Composability:**
- System layer: minimal, WSL-specific settings (stateVersion, wsl.enable, hostname)
- User layer: imports host-specific `home.nix` + sops-nix module
- Future: shared modules imported via `hosts/{hostname}/home.nix` (Phase 2)

---

### 4. Minimal WSL Configuration

**System-Level Config (per machine):**
```nix
wsl.enable = true;
wsl.defaultUser = user;
networking.hostName = hostname;
system.stateVersion = "24.05";
```

**Why Minimal:**
- WSL doesn't expose most system-level settings (kernel, boot, etc.)
- Primary work happens in user environment (home-manager)
- This satisfies "system config minimal/stable" per architecture decision
- Leaves room for future per-machine system-level divergence if needed

---

### 5. Home-Manager as Primary Tool

**Integration:**
- home-manager modules loaded at nixosSystem level
- Each machine imports `hosts/{hostname}/home.nix`
- `home-manager.useGlobalPkgs = true` to reduce duplication
- `home-manager.useUserPackages = true` for user-level package management

**Deployment Path:**
- Day-to-day: `home-manager switch --flake .#user@hostname`
- Full rebuild (rare): `nixos-rebuild switch --flake .#hostname`

---

### 6. sops-nix Integration

**Setup:**
- sops-nix available as flake input
- Integrated into all machines via `sops-nix.homeManagerModules.sops`
- Per-machine `secrets/{hostname}/` directory created for Phase 3

**Not Yet Implemented:**
- Encryption keys (generated in Phase 3)
- Actual secrets YAML files (encrypted in Phase 3)
- sops configuration in home.nix (Phase 3)
